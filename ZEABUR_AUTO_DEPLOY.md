# Zeabur 自動部署檢查

## 自動部署設置

Zeabur 預設會自動監聽 GitHub 倉庫的推送，但需要確認以下設置：

### 1. 檢查自動部署是否啟用

在 Zeabur 專案設置中：

1. 進入專案：`rating-system`
2. 點擊 **"Settings"** 或 **"設置"**
3. 找到 **"Auto Deploy"** 或 **"自動部署"** 選項
4. 確認已啟用（應該預設是啟用的）

### 2. 確認 GitHub 連接

1. 在專案設置中，找到 **"Git Repository"** 或 **"Git 倉庫"**
2. 確認已正確連接到：`sandycd570-stack/rating-system`
3. 確認分支是 `main`

### 3. 手動觸發部署（如果需要）

如果自動部署沒有觸發，可以：

1. 在專案頁面，點擊 **"Deploy"** 或 **"部署"** 按鈕
2. 或者點擊 **"Redeploy"** 或 **"重新部署"** 按鈕
3. 等待構建完成（通常需要 2-5 分鐘）

## 如何確認是否正在部署

在 Zeabur 專案頁面，您會看到：

- **"Deploying"** 或 **"部署中"** - 正在部署
- **"Building"** 或 **"構建中"** - 正在構建
- **"Live"** 或 **"運行中"** - 部署完成並運行中
- **"Failed"** 或 **"失敗"** - 部署失敗，需要查看日誌

## 建議

1. **先等待 1-2 分鐘**，看看是否自動觸發部署
2. 如果沒有自動觸發，**再手動點擊 Deploy**
3. 部署完成後，測試 `/admin` 路由是否可以訪問

---

**注意**：如果這是第一次部署，或者您剛剛連接 GitHub 倉庫，可能需要手動點擊一次 Deploy。
